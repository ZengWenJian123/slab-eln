# 实验室电子记录与信息管理系统（ELN & LIMS）UI 设计文档

**版本**：V1.0
**适用范围**：Web 管理后台 / 实验记录系统
**适配技术栈**：Next.js + App Router + TypeScript
**设计目标**：清晰、可追溯、高录入效率、低学习成本

---

## 1. UI 设计目标

本系统不是展示型官网，而是**高频录入 + 查询追溯 + 数据管理**型后台。因此 UI 目标应明确偏向以下方向：

### 1.1 核心目标

* 让实验人员能快速录入长表单
* 让管理员能快速维护字典、模板、附件和记录状态
* 让负责人能快速看懂样品来源链路、阶段进度和关键指标
* 减少误录、漏录、错录
* 保证复杂业务信息在页面中仍然清晰易读

### 1.2 设计原则

* **信息密度中高**，但分区清晰
* **操作路径短**，避免页面跳转过多
* **同类页面交互一致**
* **录入优先于花哨视觉**
* **关键状态可视化**
* **列表、详情、编辑三类页面风格统一**

## 2. 推荐前端框架与组件方案

这套给 Codex 的方案，我建议直接定死，不要让它自由发挥。

### 2.1 推荐技术组合

**主框架**

* `Next.js App Router`
* `TypeScript`

Next.js 官方文档说明，App Router 是基于文件系统的路由方案，使用了 Server Components、Suspense、Server Functions 等能力；Next.js 当前也继续以 App Router 作为主要现代路由方案。([Next.js][1])

**UI 基础**

* `Tailwind CSS`
* `shadcn/ui`
* `lucide-react`

Tailwind CSS 是 utility-first CSS framework，适合快速构建后台系统；shadcn/ui 提供大量可组合的高质量组件，并且更适合做“项目内可控组件库”，而不是黑盒依赖；Lucide React 提供可 tree-shake 的 React 图标组件，适合后台信息架构。([Tailwind CSS][2])

**表单与校验**

* `react-hook-form`
* `zod`

React Hook Form 官方文档强调其性能、灵活性和可扩展性，适合复杂表单；Zod 是 TypeScript-first 的 schema validation 库，适合统一前后端校验规则。([React Hook Form][3])

**表格与数据视图**

* `@tanstack/react-table`

TanStack Table 是 headless table/datagrid 方案，适合复杂筛选、排序、分页、列配置。([TanStack][4])

### 2.2 为什么这套组合适合你的系统

你的系统有三个典型特征：

* 长表单很多
* 表格很多
* 页面大多是“管理后台 + 实验记录详情”

因此不建议让 Codex 用偏“展示页”的组件库思路来做。
`shadcn/ui + Tailwind + React Hook Form + Zod + TanStack Table` 的好处是：

* 适合 CRUD 后台
* 组件足够成熟
* 不会被重型 design system 绑死
* 代码生成后容易继续人工维护
* 与 Next.js 配合成熟
* 对“多字段表单 + 校验 + 异常提示 + 抽屉弹窗 + 筛选表格”非常友好

### 2.3 给 Codex 的明确约束

请明确要求 Codex：

* 使用 `Next.js App Router`
* 使用 `TypeScript`
* 使用 `Tailwind CSS`
* 使用 `shadcn/ui` 作为基础 UI 组件来源
* 表单统一使用 `react-hook-form + zod`
* 数据表格统一使用 `TanStack Table`
* 图标统一使用 `lucide-react`
* 不要自行混入多个 UI 库
* 不要同时使用 Ant Design、MUI、Chakra、Element 风格混搭
* 不要生成“营销官网风”的页面

## 4. 全局视觉规范

### 4.1 视觉风格

建议风格为：

* 浅色为主
* 局部中性灰分层
* 状态色明确但克制
* 卡片圆角中等
* 不做过强阴影
* 保持实验/工程类系统气质

### 4.2 色彩建议

建议只定义少量语义色：

* 主色：蓝色系，用于主按钮、链接、当前导航
* 成功：绿色
* 警告：黄色 / 橙色
* 失败：红色
* 草稿 / 未完成：灰色
* 重点样品：紫色或琥珀色标签

### 4.3 字体层级

* 页面标题：24px / semibold
* 分区标题：18px / semibold
* 卡片标题：16px / medium
* 正文：14px
* 辅助说明：12px
* 表格正文：13px 或 14px

### 4.4 间距规范

* 页面内边距：24px
* 卡片内边距：16px ~ 20px
* 表单区块间距：24px
* 字段垂直间距：12px ~ 16px
* 表格工具栏与表格间距：16px

## 5. 通用组件规范

### 5.1 页面容器

统一使用：

* PageHeader
* FilterBar
* DataTable
* DetailSection
* FormSection
* StatusBadge
* EmptyState
* ConfirmDialog

### 5.2 状态标签规范

统一使用 Badge 表示状态：

* 草稿
* 已完成
* 处理中
* 处理成功
* 处理失败
* 已归档
* 待测试
* 重点关注

### 5.3 按钮规范

按钮只保留四类：

* 主按钮：新增 / 保存 / 提交 / 执行解析
* 次按钮：取消 / 返回 / 重置筛选
* 描边按钮：导出 / 复制参数 / 查看日志
* 危险按钮：删除 / 清空 / 彻底移除

### 5.4 表单控件规范

统一使用：

* Input：短文本 / 数值
* Textarea：长备注
* Select：字典项、状态、类型
* Combobox：可搜索下拉
* Checkbox：布尔选项
* Radio Group：二选一或少量互斥项
* Date Picker：日期
* Tabs：复杂表单的分段展示
* Dialog / Drawer：轻量新增和编辑
* Table：结构化列表数据
* Accordion：长页面分段折叠

### 5.5 提示与反馈

统一使用：

* 表单校验：字段下方红字
* 页面反馈：Toast / Sonner
* 重要风险：Alert
* 二次确认：Alert Dialog
* 加载态：Skeleton / Spinner

shadcn/ui 已提供包括 Dialog、Alert Dialog、Sidebar、Table、Tabs、Select、Toast、Skeleton、Combobox 等在内的大量成熟组件，足够覆盖本系统绝大多数页面。([Shadcn][5])

## 6.5 电弧熔炼新建页

### 页面目标

快速从成分对象生成熔炼记录，并自动计算称重和损耗率。

### 页面结构

1. 关联成分信息
2. 熔炼参数
3. 理论称重明细
4. 结果与状态
5. 附件上传

### 重点交互

* 选择成分后自动带出元素比例
* 输入目标熔炼重量后，自动生成理论称重
* 输入熔炼后质量后自动计算损耗率
* 若标记失败，则必须弹出失败原因选择

## 7. 关键交互设计规范

## 7.1 长表单设计规范

你的系统大量是长表单，所以必须统一：

* 每个表单按业务分区
* 一屏只强调一个当前分区
* 自动计算字段只读展示
* 复杂条件渲染必须动画轻量、不能跳闪
* 失败原因属于强校验字段

### 推荐方式

* 页面主内容用 Card 分区
* 复杂页用 Accordion / Tabs
* 草稿保存和正式保存可分开

## 7.2 阴性数据拦截

当用户将状态改为“失败”时：

* 立即展开失败原因区
* 失败原因 Select 为必填
* 备注建议选填，但可提示补充
* 保存时再次校验

## 7.3 自动计算字段

自动计算字段统一样式：

* 浅灰背景
* 不可手动修改
* 字段旁边显示计算说明 tooltip
* 支持复制数值

## 7.4 异常预警

异常预警不要用弹窗打断，统一用页面内 Alert：

* 黄色：预警
* 红色：硬错误
* 蓝色：说明提示

---

## 8. 列表页统一规范

所有列表页建议统一为：

1. 标题区
2. 筛选区
3. 工具栏
4. 数据表格
5. 分页

### 统一筛选交互

* 默认展示核心筛选项
* 高级筛选用 Drawer 或折叠区
* 筛选条件可一键清空
* 表格列支持显隐切换

### 统一表格交互

TanStack Table 很适合做这些能力：排序、过滤、分页、列渲染和复杂表格逻辑。([TanStack][4])

建议所有表格支持：

* 排序
* 分页
* 固定操作列
* 空状态
* 加载骨架屏
* 批量选择
* 导出当前筛选结果

## 10. 给 Codex 的实现建议

你可以把下面这段原样给 Codex：

```md
请基于 Next.js App Router + TypeScript 构建 ELN & LIMS 后台系统前端。

硬性要求：
1. 使用 Tailwind CSS 作为样式方案
2. 使用 shadcn/ui 作为基础组件体系
3. 使用 lucide-react 作为图标库
4. 所有表单统一使用 react-hook-form + zod
5. 所有列表页和数据表格统一基于 TanStack Table
6. 不要混用多个大型 UI 库
7. 页面风格以实验室后台管理系统为主，不要做营销官网风
8. 页面要优先保证录入效率、检索效率和详情追溯能力
9. 所有页面采用统一后台布局：左侧 Sidebar + 顶部 Header + 主内容区
10. 优先输出可复用组件：PageHeader、FilterBar、DataTable、StatusBadge、FormSection、DetailSection
```
