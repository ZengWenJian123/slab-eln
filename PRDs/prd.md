实验室电子记录与信息管理系统 (ELN & LIMS) PRD
1. 项目概述

本项目旨在开发一套适用于小型实验室的 Web 端电子记录与信息管理系统。系统聚焦于合金非晶丝实验流程的数字化管理，覆盖从 成分设计 → 电弧熔炼 → 拉丝及玻璃退除 → 后处理 → 性能测试 的全生命周期记录。

系统核心目标如下：

实现实验过程的标准化录入与结构化存储
实现样本、批次、原始数据、分析结果之间的严格溯源
降低实验记录遗漏、命名混乱、参数不统一等问题
支持实验原始文件上传、结果提取、数据回填与看板展示
在不增加复杂部署负担的前提下，优先采用 Next.js + MySQL 完成系统实现，适合小团队持续迭代和维护
2. 产品定位与使用对象
2.1 产品定位

部署简单

对实验样本与批次管理足够严格
便于后续让 AI / Codex 基于 PRD 快速生成代码

3. 核心系统架构
3.1 技术架构
前端：Next.js
后端：优先使用 Next.js 内置服务能力
Route Handlers
Server Actions
服务端渲染与服务端数据获取
数据库：MySQL
3.2 架构原则
单体化优先：前后端尽量放在同一个 Next.js 项目中，不拆分独立后端服务
降低部署复杂度：适合直接部署到宝塔面板等常见环境
功能优先于技术炫技：避免引入额外中间件、消息队列、对象存储等新架构
可维护性优先：表结构、接口、页面流程应尽可能直观，方便后续修改
3.3 文件存储策略

系统上传的原始 CSV/TXT/图片/PDF 文件，先存储在服务器本地目录，路径写入 MySQL 进行索引和关联，不引入额外对象存储服务。

建议目录规则：

/uploads/raw-data/
/uploads/attachments/
/uploads/qrcode/

数据库中保存：

文件原始名
文件存储路径
文件类型
文件大小
上传时间
关联样品 UUID
4. 核心设计原则
UUID 作为唯一主键，显示名称仅用于沟通
每个工序独立记录，但彼此可追溯
失败记录必须可保存，但必须说明原因
所有关键计算尽量系统自动完成，减少手算
所有关键下拉项、枚举项统一管理，避免录入混乱
页面交互以“实验人员快速录入”为中心，而不是 ERP 式复杂管理
5. 核心功能模块划分
5.1 样本追踪与命名引擎（UID System）
5.1.1 唯一标识

每一个物理实体在系统中生成时，立刻赋予一个唯一的 UUID v4 作为主键。该主键不承载业务语义，仅用于数据库关联与溯源。

适用对象包括但不限于：

合金成分
熔炼锭子
拉丝批次
裸丝 / 包覆丝样本
2cm 焊板样本
后处理样本
性能测试记录
5.1.2 显示别名引擎

系统根据用户输入规则自动生成人类可读的语义化名称，仅用于展示、打印、检索和实验沟通。

推荐规则：
[成分代号]-[熔炼批次]-[拉丝批次]-[丝状态+直径]-[后处理代号]-[样本序号]

示例：
H8-M0325-L0330-GR32-DC20-1

含义：

H8：成分代号
M0325：3 月 25 日熔炼批次
L0330：3 月 30 日拉丝批次
GR32：裸丝 32 μm
DC20：20mA 直流退火
1：样本序号
5.1.3 条码 / 二维码

系统支持为样品生成二维码或条码，并支持打印标签，用于：

样本袋粘贴
焊板标识
数据上传时扫码绑定
5.1.4 命名冲突保护

若展示别名重复，系统自动追加流水尾缀或提示用户调整，避免人工命名混乱。

5.1.5 样本关系树

系统提供“来源链路”展示：

当前样本来自哪个拉丝批次
拉丝批次来自哪个熔炼批次
熔炼批次对应哪个成分设计
当前样本有哪些后处理和测试记录

5.2.1 工序一：成分设计（Alloy Design）
输入项

默认元素库可包含：

Co
Fe
Si
B
Cr
Cu
Nb

验证规则

5.2.2 工序二：电弧熔炼（Arc Melting）
关联

选择已有成分代号，自动带出成分信息。

输入项
熔炼日期
目标熔炼重量（必填）
各元素理论称重结果（系统自动计算）
熔炼后锭子质量
熔点（选填）
实验备注
计算逻辑

系统根据目标熔炼重量和成分原子百分比，自动反算各元素应称重质量。

自动计算：
熔炼损耗率 = (目标熔炼重量 - 熔炼后锭子质量) / 目标熔炼重量 × 100%

输出
熔炼批次号：M + MMDD
例如：M0325
辅助功能
自动标记损耗率是否偏高
支持上传称量照片
支持保存“中途中断/失败”记录
支持批量打印该批次关联标签
5.2.3 工序三：拉丝及玻璃退除（Melt Spinning & Glass Removal）
关联

选择熔炼批次号，自动带出成分代号及基础信息。

输入项
玻璃管直径
放料质量
拉丝温度
卷绕转速（rpm）
冷却水温度
负压（kPa）
包覆丝直径（μm）
裸丝直径（μm）
玻璃腐蚀时间
形态状态（GC / GR）
实验备注
扩展校验
若拉丝温度明显低于熔点，系统高亮警告
若裸丝直径大于包覆丝直径，禁止提交
若关键字段为空，禁止保存
自动计算

玻璃层厚度 = (包覆丝直径 - 裸丝直径) / 2

输出
拉丝批次号：L + MMDD
例如：L0330
分支设定

定义样品形态：

GC：玻璃包覆
GR：裸丝

提供选项：

是否进行磁学等表征（Br / Bs / Hc）？
勾选后可直接跳转到性能记录模板。
辅助功能
保存常用设备参数组合，便于下次复用
显示同成分历史拉丝参数对比
自动提醒“本次直径偏离历史均值”
5.2.4 工序四：样品制备态及后处理（Post-treatment）
前置判断

选择样品批次后，系统自动带出：

当前状态（GC / GR）
裸丝直径
关联拉丝批次

系统询问：
“是否已焊接于 2cm 焊板？”

是：视为特定制备态，冻结部分拉伸相关选项
否：开放全部退火选项
后处理参数矩阵

支持如下类型：

温度退火
温度（℃）
保温时间（h）

电流退火

类型（直流 / 脉冲）
电流密度（A/mm²）
通电时间（min）
占空比（%）
频率（Hz）

自动计算：
电流大小(mA) = 电流密度 × π × (裸丝直径/2)^2

拉伸退火

拉伸配重（g）
拉伸时间（min）

自动计算：
拉伸力(MPa) = (拉伸配重 × 9.8) / 截面积

复合退火
温度 + 拉伸
电流 + 拉伸
磁场退火（2cm 专属）
磁场方向（轴向 / 横向）
磁场大小（Oe）
输出

生成后处理代号，例如：

TA500
DC20
PA15
MF-Axial
辅助功能
根据处理类型自动拼接后处理简称
允许从已有后处理记录复制参数
自动校验输入值是否超常规范围并提示
记录“前后状态对比备注”
5.2.5 工序五：性能检测与数据绑定（Data Pipeline）
原始数据绑定

用户上传测试设备生成的 CSV / TXT / 图片 / PDF 文件时，需要扫描或输入样品 UID / 别名。

系统完成以下操作：

文件上传至服务器本地目录
在 MySQL 中记录文件路径与样品 UUID 的绑定关系
生成原始数据记录表
支持后续重新解析
自动化清洗入口

系统提供“执行解析”按钮，调用已有 Python 脚本处理原始数据文件。

脚本处理目标

从原始数据中提取关键测试结果，例如：

GMI ratio 最大值
特征阻抗
峰值位置
磁滞参数
其他结构化指标
结果存储

解析后的指标写回 MySQL，与样品性能记录表绑定，用于后续看板展示与检索。

处理状态

无需引入任务队列，也应提供基础状态字段：

待处理
处理中
处理成功
处理失败

若失败，记录：

错误摘要
失败时间
可重新执行
辅助功能
支持原始文件重复上传提醒
支持重新解析旧文件
支持查看解析日志摘要
支持手动补录解析失败后的关键指标

12. 对原 PRD 中不适合当前架构的部分修正说明

以下内容建议从 PRD 中替换或弱化：

12.1 MinIO

原文写“自动存入 MinIO”，但你当前要求是不引入新架构。
建议改为：

原始文件上传到服务器本地目录
文件路径记录到 MySQL
后续如果规模扩大，再考虑升级到对象存储
12.2 MongoDB

原文写“结构化指标写回 MongoDB”，但当前数据库只保留 MySQL。
建议统一改为：

原始记录、解析结果、统计指标全部写入 MySQL
12.3 Dynamic Schema Engine

原文提到 JSON Schema 动态表单引擎。这个对第一版来说有点超。

7. 功能需求
7.1 样本追踪与命名引擎（UID System）
7.1.1 功能目标

建立稳定的样本标识体系，解决样品命名混乱、人工命名变化导致数据断链的问题。

7.1.2 功能说明
1）UUID 主键机制

每个物理实体在系统中创建时，自动生成一个 UUID v4 作为唯一主键。
UUID 不承载业务语义，仅用于系统内部关联。

适用对象包括但不限于：

合金成分
熔炼批次
拉丝批次
裸丝 / 包覆丝样品
焊板样品
后处理样品
性能测试记录
2）显示别名生成

系统根据规则自动生成可读性名称，用于展示、打印、搜索和人工沟通。

推荐格式：
[成分代号]-[熔炼批次]-[拉丝批次]-[丝状态+直径]-[后处理代号]-[样本序号]

示例：
H8-M0325-L0330-GR32-DC20-1

3）命名稳定性要求

即使展示别名因业务参数修改而变化，系统底层 UUID 关联关系必须保持不变。

4）条码 / 二维码

系统支持根据样品 UUID 或展示别名生成条码 / 二维码，用于打印和扫码录入。

5）来源关系链

在样品详情页中，应可查看当前样品的来源链路，包括：

成分来源
熔炼批次
拉丝批次
后处理记录
性能测试记录
原始文件与解析结果
7.2 工序一：成分设计（Alloy Design）
7.2.1 功能目标

标准化记录合金成分设计信息，为后续熔炼和样本生成提供基础对象。

7.2.2 输入项
元素库多选
各元素原子百分比
设计备注

默认元素库：

Co
Fe
Si
B
Cr
Cu
Nb
其他由管理员维护的元素
7.2.3 校验规则
所有元素原子分数之和必须严格等于 100%
不允许存在重复元素
任一元素比例不能为空

若不满足规则，前端阻止提交并提示错误。

7.2.4 输出
自动生成成分代号，如 H1、H2、H3
7.2.5 辅助功能
显示百分比实时合计结果
支持复制已有成分作为新模板
支持收藏常用成分配方
7.3 工序二：电弧熔炼（Arc Melting）
7.3.1 功能目标

记录熔炼过程关键参数和结果，并关联成分设计对象。

7.3.2 关联关系

需选择已有成分代号，自动带出成分信息。

7.3.3 输入项
熔炼日期
目标熔炼重量（必填）
熔炼后锭子质量
熔点（选填）
实验备注
可上传附件（称量照片等）
7.3.4 自动计算

系统根据目标熔炼重量和元素比例自动反算各元素应称重质量。

自动计算：
熔炼损耗率 = (目标熔炼重量 - 熔炼后锭子质量) / 目标熔炼重量 × 100%

7.3.5 输出
熔炼批次号：M + MMDD
示例：M0325
7.3.6 异常与失败

若熔炼失败，必须填写失败原因后方可保存。

失败原因可从字典项中选择，例如：

坩埚破裂
熔炼不充分
材料损耗异常
设备故障
其他
7.3.7 辅助功能
对高损耗率进行提示
支持批量打印批次标签
支持记录中断状态
7.4 工序三：拉丝及玻璃退除（Melt Spinning & Glass Removal）
7.4.1 功能目标

记录拉丝和玻璃退除过程参数，生成对应拉丝批次和样品对象。

7.4.2 关联关系

选择熔炼批次号，自动带出成分代号与相关基础信息。

7.4.3 输入项
玻璃管直径
放料质量
拉丝温度
卷绕转速（rpm）
冷却水温度
负压（kPa）
包覆丝直径（μm）
裸丝直径（μm）
玻璃腐蚀时间
形态状态（GC / GR）
备注
7.4.4 自动计算

玻璃层厚度 = (包覆丝直径 - 裸丝直径) / 2

7.4.5 校验规则
裸丝直径不得大于包覆丝直径
关键字段不能为空
若拉丝温度低于材料熔点，系统进行高亮预警
7.4.6 输出
拉丝批次号：L + MMDD
示例：L0330
7.4.7 分支逻辑

形态状态定义：

GC：玻璃包覆
GR：裸丝

提供选项：

是否进行磁学等表征（Br / Bs / Hc）
勾选后可直接跳转至性能记录模板。
7.4.8 辅助功能
支持保存常用参数模板

7.5 工序四：样品制备态及后处理（Post-treatment）
7.5.1 功能目标

记录样品后处理方式和参数，并生成后处理后的样品记录。

7.5.2 前置判断

选择样品批次后，系统自动带出：

直径信息
关联拉丝批次

系统需询问：

是：冻结部分拉伸相关选项
否：开放全部可选项
7.5.3 支持的后处理类型
1）温度退火
温度（℃）
保温时间（h）
2）电流退火
类型（直流 / 脉冲）
电流密度（A/mm²）
通电时间（min）
占空比（%）
频率（Hz）

自动计算：
电流大小(mA) = 电流密度 × π × (裸丝直径 / 2)^2

3）拉伸退火
拉伸配重（g）
拉伸时间（min）

自动计算：
拉伸力(MPa) = (拉伸配重 × 9.8) / 截面积

4）复合退火
温度 + 拉伸
电流 + 拉伸
5）磁场退火（2cm 专属）
磁场方向（轴向 / 横向）
磁场大小（Oe）
7.5.4 输出

自动生成后处理代号，例如：

TA500
DC20
PA15
MF-Axial
7.5.5 异常与失败

若后处理失败，必须填写失败原因与说明。

7.5.6 辅助功能
